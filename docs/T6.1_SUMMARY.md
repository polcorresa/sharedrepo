# T6.1 Summary: Frontend Shell

## Overview
Implemented the basic frontend structure, routing, and the repository entry flow (Repo Gate).

## Components

### 1. Routing (`apps/web/src/router.tsx`, `apps/web/src/routes/tree.ts`)
- **TanStack Router**: Configured with `RootLayout`.
- **Routes**:
  - `/`: Landing Page.
  - `/$slug`: Repo Gate (Create/Login/Editor).

### 2. Pages
- **LandingPage** (`apps/web/src/pages/LandingPage.tsx`):
  - Explains the product.
  - Form to enter a slug and navigate to `/$slug`.
- **RepoGate** (`apps/web/src/pages/RepoGate.tsx`):
  - Fetches repo status (`GET /api/repos/:slug/status`).
  - Shows **Create Form** if repo is available.
  - Shows **Login Form** if repo exists.
  - Handles authentication via `POST /api/repos` or `POST /api/repos/:slug/login`.
  - Renders `RepoEditor` if authenticated (metadata fetch succeeds).

### 3. Components
- **RepoEditor** (`apps/web/src/components/RepoEditor.tsx`):
  - Placeholder for the main editor UI.
  - Rendered when user is authenticated.
- **RootLayout** (`apps/web/src/components/layout/RootLayout.tsx`):
  - Basic shell for the app.

### 4. Utilities
- **API Client** (`apps/web/src/lib/api.ts`):
  - Wrapper around `fetch` with `credentials: 'include'` for cookie handling.
  - Handles base URL and error parsing.
